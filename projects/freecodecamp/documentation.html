Introduction

This is the introduction section. This section covers much of the technical jargon used throughout this documentation, with short descriptions and examples.

DDI
DNS, DHCP, IP Address Management. These are the services that manage the IP Address resources for computers to be able to look up the hostname or IP Address destination in order to send and receive data.
DNS
Domain Name System/Service. This is the server or service that provides the directory of FQDNs (hostnames) and IP Addresses. Each DNS Server stores this data for its respective domain(s) that it manages. There are several kinds of DNS servers, including Primary, Secondary, and Caching.
DHCP
Dynamic Host Control Protocol. This is the service or server that provides IP Address assignments to computers that join the network. Every computer on the network requires an IP Address in order to communicate with other computers and servers in order for the user to perform their job duties.
IP Address
Internet Protocol Address. All computers now "speak" to each other over the Internet using the TCP/IP protocol. An IP Address is a set of assigned numbers. There are two versions of IP Address numbers being used today, IPv4 and IPv6.
IPv4
IP Version 4 is the initially available addressing scheme for hosts on the Internet, and is still pretty much the primary one. Example: 10.1.1.4
IPv6
IP Version 6 has been around for quite a while, however, has not gained enough traction to become the primary addressing scheme. There are enough addresses for all the devices in the world, several times over, however, not all computer and networking equipment can handle the extra resource requirements needed. Example: 2607:fb90:8c4:bc0a:ba9c:ac90:74aa:b2e7

Domain Name System

This is the section on DNS. It will be awesome when I complete it.

DNS works like a root system for a tree. The hostname is always on the far left, and ends with a .on the far right. For example, homer.simpson.net. indicates homer is the hostname, simpson.net is the primary (or often called TlD = Top-level-domain), and ends with ..

When you first type a web query (for example www.aol.com) into the address field of your browser, your computer has to go look up the IP Address of the server(s) serving up the web content to request and display in your browser.
